# 分页功能改进说明

## 改进内容

### 1. 新增API函数
在 `frontend/src/lib/api.ts` 中新增了：
- `getRfcsPaginated(page, perPage)` - 获取分页的RFC列表

### 2. 创建通用分页组件
创建了 `frontend/src/components/pagination.tsx`，具有以下特性：
- **智能页码显示**：当页数较多时自动显示省略号
- **响应式设计**：支持深色模式
- **无障碍支持**：包含适当的ARIA标签
- **灵活配置**：支持自定义查询参数

#### 分页组件功能
- 显示当前页和总页数
- 上一页/下一页按钮
- 智能页码按钮（最多显示7个）
- 页码过多时显示省略号
- 禁用状态处理（首页/末页）

### 3. 更新RFC列表页面
- `frontend/src/app/rfcs/page.tsx` - 使用Suspense包裹，支持SSR
- `frontend/src/components/rfc-list.tsx` - 改为客户端组件，支持分页交互
  - 显示总数统计
  - 分页加载
  - 加载状态显示
  - 错误处理

### 4. 更新搜索页面
- `frontend/src/app/search/page.tsx` - 使用新的通用分页组件
- 保持查询参数（q）在分页导航中

### 5. 更新首页
- `frontend/src/app/page.tsx` - 使用Suspense包裹RfcList组件

## 使用示例

### RFC列表页面
```
/rfcs          - 第1页（默认）
/rfcs?page=2   - 第2页
/rfcs?page=10  - 第10页
```

### 搜索页面
```
/search?q=http&page=1   - 搜索"http"的第1页
/search?q=http&page=2   - 搜索"http"的第2页
```

## 分页参数

- **默认每页数量**：20条
- **页码从1开始**
- **后端API支持参数**：
  - `page` - 页码（从1开始）
  - `per_page` - 每页数量（默认20，最大100）

## 技术细节

### 客户端分页
- 使用 `useSearchParams` 获取URL参数
- 使用 `useEffect` 监听页码变化
- 使用 `useState` 管理数据和加载状态
- 必须用 `Suspense` 包裹以支持SSR

### 服务端分页
- 搜索页面保持服务端渲染
- 数据在服务端获取
- 分页导航通过Link实现

### 页码显示逻辑
```
总页数 ≤ 7：显示所有页码
1 2 3 4 5 6 7

当前页靠前（≤ 3）：
1 2 3 4 5 ... 20

当前页靠后（≥ 总页数-2）：
1 ... 16 17 18 19 20

当前页在中间：
1 ... 9 10 11 ... 20
```

## 用户体验改进

1. **加载状态**：显示加载动画，避免空白
2. **错误处理**：显示友好的错误提示
3. **总数显示**：在列表顶部显示"共X个RFC文档"
4. **页码信息**：分页组件右侧显示"当前页/总页数"
5. **禁用状态**：首页禁用"上一页"，末页禁用"下一页"
6. **键盘导航**：所有按钮可通过键盘访问

## SEO优化

- 分页URL清晰可读：`?page=2`
- 保持查询参数：`?q=keyword&page=2`
- 支持服务端渲染
- 使用语义化HTML标签
- 包含适当的ARIA标签
