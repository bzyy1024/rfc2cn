## 2. RFC详情页上一个/下一个导航功能

### 新增内容
1. **API函数** (`lib/api.ts`):
   - 新增 `getAdjacentRfcs()` 函数
   - 从RFC列表中查找当前RFC的上一个和下一个
   - 返回相邻RFC的信息，如果不存在则返回null

2. **UI组件** (`rfc/[number]/page.tsx`):
   - 在导航栏添加"上一个"和"下一个"按钮
   - 按钮显示在页面顶部右侧
   - 包含鼠标悬停时显示RFC标题的提示
   - 如果没有上一个或下一个，显示禁用状态

### 测试步骤
1. 访问任意RFC详情页（例如：`/rfc/1`）
2. 检查页面顶部是否显示导航按钮
3. 点击"下一个"按钮，验证是否跳转到下一个RFC
4. 点击"上一个"按钮，验证是否跳转到上一个RFC
5. 访问第一个RFC，验证"上一个"按钮是否禁用
6. 访问最后一个RFC，验证"下一个"按钮是否禁用
7. 鼠标悬停在按钮上，验证是否显示RFC标题提示

### 预期结果
- 导航按钮正确显示在页面顶部
- "上一个"按钮显示在左侧，"下一个"按钮显示在右侧
- 按钮样式在hover时有过渡效果
- 第一个RFC的"上一个"按钮显示为禁用状态
- 最后一个RFC的"下一个"按钮显示为禁用状态
- 点击按钮后正确跳转到相应的RFC页面
- 悬停时显示完整的RFC标题

4. **面包屑导航**：在详情页添加面包屑显示当前位置
5. **搜索历史**：记录用户的搜索历史

这些改进不是必需的，当前实现已经满足基本需求。
